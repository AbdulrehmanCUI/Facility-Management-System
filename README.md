Recently, I worked on a Facility Device Management System in Java as part of my learning journey, and it turned out to be a great way to really understand OOP concepts beyond just theory.

The idea was simple: create a system that can manage different devices in a facility like projectors, climate control systems, and cameras. But while building it, I got to implement some important concepts in a practical way.

I designed a base class for general device properties (like name, location, and status), and then extended it for specific devices. Each device behaves differently when activated or deactivated, which helped me understand inheritance and method overriding much better.

The system allows you to:
Add new devices.
Turn devices ON/OFF.
View all devices and their details.
Compare devices based on their attributes.

I also implemented a menu-driven console interface so the user can interact with the system step by step, along with basic exception handling to make it more stable.

This project helped me connect the dots between concepts like encapsulation, polymorphism, and real-world problem solving.
Next step: thinking of improving this by adding a graphical interface and maybe connecting it to a database.
